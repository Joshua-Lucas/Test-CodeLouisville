Why use semantic HTML??

Three reasons why semantic HTML is critical for today's web.

  1. User-friendliness: It provides for cleaner written code. It helps both browser and developer understand your code.
  2. Acessibility: Semantic code allows your web page to be accessible. The web is for everyoneat its core the web is accessible, it is we as developers that make it unaccessible. 1 in 5 people in the world have a disability, and we want them to have a great user experience on the web.
  3. SEO(Search Engine Optimization) Semantic HTML makes it easier for the computers that crawl your site in order to rank you website. Also SEQ is now giving more focus to accessiblilty when ranking your site.

Examples of semantic HTML

The structure of the document

You should use the header, main, and footer elements to break up your page.

The Header Element

The header specifies the header of the document or section of the page. You are able to have multiple headers in the document, but you cannot nest it in a footer,address or another header element.

Elements normally in the header.

  • Navigation
  • Logo or icon
  • one or more heading elements
  • authorship info

The Main Element

The main element specifies the main content of the document.

The Footer Element

The footer defines the footer for the document or section. This means you can have multiple footers in one document.

Common element in a footer

  • copy right info
  • contact info
  • another navigation section
  • back to top button

Section Element

This element defines a section in your document. W3C defines a section as "a thematic grouping of contented, typically with a heading"
